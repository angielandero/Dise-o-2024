Project 3

The image displays a basic form on a webpage designed for adding new items to a collection or catalog, possibly for a retail system or warehouse inventory. The form includes the following fields: Item Code: A numeric identifier for the product. Item Name: A text field for entering the product name. Description: A larger text area for providing a brief description of the item. Cost Price: The original price at which the item was acquired. Retail Price: The price at which the item will be sold. Stock Quantity: A number indicating how many units of the item are available. Date Added: A date picker for specifying when the item was added to the inventory. Product Image: An upload button for attaching an image of the product. A Submit button is located at the bottom to finalize and save the entry. The page likely belongs to a system designed for managing a retail store, warehouse, or similar inventory-driven business. It has a clean, user-friendly layout with intuitive input fields.

Project 4

This form allows users to input details about an item they wish to add to their inventory. It prompts the user to provide: Item Code: A unique numeric identifier for the item. Item Name: The product's name or title. Cost: The purchase price of the item. Sale Price: The price at which the item will be sold. Sale Percentage: The discount percentage applied to the sale price, if applicable. Date Added: The date the item was entered into inventory, using a date picker. Item Image: An option to upload a picture of the item for visual reference. At the bottom of the form, a Save button is available to submit and store the entered information in the system. This form is part of an inventory management system, simplifying the process of tracking and managing items.
